# LBRC Website Project

New website build for Lil Big Run Club — agency brief issued April 2026 to support a rebrand and establish a digital home.

## Summary

[[lil-big-run-club]] issued an agency brief in April 2026 to commission a new website aligned with their rebrand. The project prioritises mobile-first design, easy non-technical content management, and a clear path for new runners to discover and attend their first run. Future phases include merch sales, memberships, and expanded partnerships.

## Details

### Goals

1. Establish a digital home reflecting the refreshed brand identity
2. Make it effortless for new runners to find LBRC, understand what's offered, and show up to their first run
3. Keep ongoing maintenance minimal — easy for non-technical team members to update
4. Support future growth: merch sales, partnerships, memberships
5. Mobile-first performance — audience checks run times on their phone, often on the go

### Proposed Sitemap

| Page | Key Features |
|------|-------------|
| **Homepage** | Hero with value proposition, next upcoming run (date/time/location/CTA), Instagram feed embed, quick links (New Here? / Schedule / Merch / Newsletter), rotating member spotlight |
| **New Here? / Your First Run** | What to expect, pace and distance options, "no one gets left behind" messaging, what to bring, meeting point, parking/PT info, FAQ |
| **Run Schedule** | Weekly recurring runs with day/time/location/distance/pace, embedded route maps with meeting point pins and PT stops, Strava integration, wet weather policy, winter darkness advisory |
| **Events** | Upcoming special events and social runs, tied to Melbourne race calendar, RSVP/registration links, past events archive via Instagram embed |
| **About Us / The Team** | Club story, mission, values, coach and run lead bios (photo, name, bio, fun fact) in single-page grid format; dedicated sub-pages only if 4+ coaches |
| **Merch / Shop** | Shopify Lite or Buy Button embed, pre-orders and limited drops, member discount codes via email, sizing guides, shipping and returns info |
| **Partners** | Logo grid of current partners/sponsors, short description per partner (especially post-run café partners), contact CTA for prospective partners |
| **Contact** | Contact form (name, email, message, enquiry type dropdown); social links; no raw email address published |
| **Policies & Legal** | Website T&Cs, privacy policy, liability waiver / participation disclaimer (insurer requirement — legal professional wording required), code of conduct |

### Feature Priority Matrix

**Must Have (launch)**
- New Here? page — highest-conversion page for new members
- Run schedule with maps — core utility for visitors
- Mobile-first responsive design — non-negotiable for this audience
- Instagram feed embed — auto-updates, replaces standalone gallery
- Newsletter signup via Mailchimp — free tier covers 500 contacts
- Contact form — simple enquiry form, no raw email
- Google Analytics 4 — free; event tracking set up at launch
- Policies & liability waiver — legal requirement for participation
- Coach/team bios — single-page grid format
- Events page — tied to Melbourne race calendar
- Partner logo page — includes prospective partner CTA

**Nice to Have**
- Shopify Buy Button for merch (~$5–15/mo; embed into any platform)
- Member spotlight — rotating homepage feature, updated monthly
- Strava integration — adds community credibility
- Quick-filter pills on schedule — e.g. "Morning runs", "Under 5km", "Dog-friendly"

**Phase 3 (post-launch)**
- Members-only area — only if charging fees; adds login overhead

**Not Recommended**
- Anonymous Q&A submissions — requires moderation; static FAQ is safer

### Maintenance Philosophy

Guiding principle: every feature must pass the "will someone actually update this?" test. Features that will go stale hurt credibility more than not having them.

| Proposed Feature | Recommendation | Reason |
|-----------------|----------------|--------|
| Standalone photo gallery | Replace with Instagram embed | Auto-updates; no duplicate uploads |
| Dedicated coach bio sub-pages | Single team grid page | Less to maintain; expand later if needed |
| Live Q&A with anonymous submissions | Replace with static FAQ | Zero moderation; update quarterly |

### Tech & Platform

| Area | Decision / Question |
|------|-------------------|
| CMS/Platform | TBD — agency to recommend (Squarespace, Webflow, WordPress, or custom); must allow non-technical updates |
| E-commerce | Shopify Lite or Shopify Buy Button; must support pre-orders and discount codes |
| Newsletter | Mailchimp (or equivalent); free tier for 500 contacts; welcome email template setup included in build |
| Analytics | GA4 with event tracking: newsletter signups, merch clicks, schedule views, contact submissions; basic reports for partners |
| Members area | Not at launch; agency to advise on difficulty of adding later on chosen platform |

### Operational Questions for Agency

The brief asks the agency to address in their proposal:
- Platform/hosting recommendation, non-technical update capability, and ongoing hosting costs
- E-commerce setup, pre-order support, discount codes, and monthly costs
- Mailchimp integration and welcome email template as part of the build
- GA4 event tracking setup and basic partner reporting
- Members-only area: how difficult to add post-launch on the recommended platform
- Handover plan, maintenance retainer offering, platform updates/security, itemised ongoing costs
